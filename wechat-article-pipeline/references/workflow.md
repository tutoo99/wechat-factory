# 公众号流水线参考

## 一、推荐的完整链路

### 从零开始

1. **加载人设配置**：读取 `wechat-factory/persona-<账号>.yaml`
2. **推荐框架并让用户选编号**：先跑 `framework_flow.py` 输出 Top 3，再让用户选 `1/2/3`
3. **准备大纲指令包**：选中编号后跑 `framework_flow.py --code N --prepare-outline`，脚本生成 `*.outline.prompt.md`，不调用 LLM
4. **当前 AI CLI 生成大纲**：在哪个 CLI 会话里发起写作任务，就由当前会话读取 `outline.prompt.md` 并生成 `*.outline.yaml`
5. **回传大纲并召回素材**：跑 `framework_flow.py --code N --outline-file <outline.yaml>`，脚本校验大纲、按节召回素材、生成新版写作 prompt
6. 定位文章类型
7. 确定情绪按钮 / 观点核心
8. 写正文初稿（以人设中的叙述者身份写作）
9. 做轻度降 AI
10. 违禁词排查（`banned-word-guard`），发现并替换违禁词
11. 输出发布素材

### 已有初稿

1. 判断稿件问题在哪
2. 优先做人味增强
3. 违禁词排查（`banned-word-guard`）
4. 再出标题摘要封面

## 二、推荐默认产物

一套完整交付通常包括：

- 正文终稿
- 标题 3 到 9 个
- 摘要 1 到 3 个
- 导语 1 到 2 个
- 封面主标题 / 副标题
- 如有需要，再补首图短句

## 三、正文写作建议

如果是情绪文 / 观点文，正文优先按下面的骨架来：

- 开头钩子
- 冲突建立
- 剧情升级
- 真相揭露
- 观点提升
- 解决方案 / 提醒
- 收尾金句

### 小标题建议

如果做公众号版，建议给正文加 3 到 6 个小节标题。  
小节标题最好写成：

- `01｜麻烦往往从一句实话开始`
- `02｜他不是来照顾你的，是来试探你的`
- `03｜很多晚年的麻烦，都是从“心软”开始的`

不要只写：

- `01`
- `02`
- `03`

纯编号只有分段感，没有信息钩子，看起来也更像半成品。

## 四、后处理建议

写完以后，不要立刻交稿。  
先快速检查：

- 后半段有没有太像模板
- 金句是不是太密
- 结尾是不是太圆
- 有没有“对，但不像真人”的地方

如果有，就做一次轻度降 AI。

## 五、推荐调用话术

### 一句话全流程

- “用 `wechat-article-pipeline` 围绕 X 做一篇完整公众号稿。”
- “围绕 X 写一篇公众号文，写完后自动降 AI，并补标题摘要封面。”
- “把这个主题做成公众号可发版，从正文到发布素材一起给我。”

### 指定情绪文

- “围绕 X 写一篇公众号情绪文，先按爆款结构写，再做轻度降 AI。”
- “做一篇观点文，保留传播感，但最后润得更像真人写的。”

### 指定已有初稿

- “这是一篇初稿，用 `wechat-article-pipeline` 把它处理成可发版。”
- “别重写太多，先润色，再补标题摘要封面。”

## 六、推荐默认口径

当这个 skill 被调用时，默认遵循：

- 写稿重传播
- 大纲先行，先想清楚每节核心观点再写正文
- 当前 AI CLI 负责生成大纲，脚本只负责准备 prompt、校验 outline、按节召回和组装写作 prompt
- 润色重人味
- 包装重点击和适配公众号

最终目标不是“写得很完整”，而是：

- 读者愿意点开
- 能顺着读下去
- 觉得像真人写的
- 可以直接发
