## Framework

- id: `tech_product_conversion_affiliate`
- name: 教程案例 + 佣金转化型
- summary: 适合"用详细教程和真实案例证明某个工具/平台好用，文中多处嵌入购买链接赚佣金"的文章。读者因为拿到了可复制的实操方案而信任推荐。
- hook_pattern: 引用上篇爆款数据建立可信度，再用"读者问得最多的问题"切入，给出"XX 2.0 省钱版/升级版"的定位，让读者觉得这篇是应需而来。
- ending_pattern: 用"三步开始"把整个教程收束成简单动作，再给购买链接和套餐建议，让读者从"学到东西"自然过渡到"下单"。

### Required Materials
- 一个可复制的完整操作流程（读者能照着做）
- 至少 2 个真实 Case 证明方案可行
- 一个具体的推荐产品，有明确的购买链接
- 前后对比数据（成本/效率/效果）

### Section Flow
- 开头：引用上篇数据 + "读者最关心的问题" + 本篇定位（升级版/省钱版/补充版）
- 产品露出（轻量）：顺带提一句推荐产品，带链接，不展开，"但便宜不是理由，能干活才是"做转折
- 架构/方案讲解：用清晰的结构图或列表说明整体方案设计
- 搭建教程：给可直接复制粘贴的 prompt / 配置 / 代码，手把手教
- Case 验证（2-3个）：每个 Case 统一结构（痛点 → 指令 → 执行过程 → 效果总结），证明方案可行
- 收束总结：算总账（成本对比/效率对比），给出一句话结论
- 转化收口："三步复刻"式行动指引 → 产品购买链接 → 套餐选择建议

### Constraints
- 教程部分要有可直接复制的代码/prompt/配置，不能只讲思路不给东西
- Case 要有真实细节（具体指令、具体输出、具体耗时），不能泛泛而谈
- 产品链接分散嵌入（开头、中间、结尾各一处），不要只在底部集中放
- 付费转化要自然，产品是教程方案的"基础设施"，不是额外推销
- 结尾的套餐建议要帮读者做选择（轻度用选X，重度用选Y），降低决策成本
