# 技术密记预览

这套风格适合把更多信息压进更短的可视高度里，像工程笔记，而不是像杂志封面。

## 背景界定

当文章本身就包含大量步骤、表格、判断标准时，过多的视觉留白会让读者来回滑动，影响“收藏后二刷”的效率。

### 适用场景

更适合复盘、排障、配置清单、经验沉淀。

> 技术号的高级感，很多时候不是颜色高级，而是信息组织得让人觉得“省时间”。

- 重点更密
- 留白更克制
- 结构更像笔记

1. 问题定义
2. 关键排查
3. 修复动作
4. 复用判断

```bash
pnpm install
pnpm run build
pnpm run deploy
```

| 维度 | 策略 |
|------|------|
| 段落 | 更短更密 |
| 小标题 | 更直接 |
| 代码块 | 更像终端 |
