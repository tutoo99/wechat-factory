# 飞书CLI对话式安装流程（真实复盘）

> 这是真实的用户与AI对话式安装飞书CLI并自动建表的全过程。
> 来源：2026.4.29 对话复盘，一字未改。

## 流程回顾

### Step 1: 安装

用户发了一个GitHub链接：
> 帮我装一下所有的东西：https://github.com/larksuite/cli/blob/main/README.zh.md

AI自行完成：打开终端、执行安装命令、处理依赖。

### Step 2: 创建飞书应用

AI给用户一个链接，用户打开后扫码登录飞书，创建了一个叫"飞书CLI"的应用。

用户告诉AI：
> 飞书应用已经建好了。

### Step 3: 完成授权

AI给用户另一个授权链接，用户在飞书上完成应用授权。

用户告诉AI：
> 我已经完成授权。

### Step 4: AI自动建表

用户说：
> 我想把拆解的爆文文章的拆解的关键信息，存入飞书多维表格，但是我还没新建多维表格

AI自行：研究飞书多维表格使用方法 → 创建表格 → 设计17个字段（文章标题、公众号、文章链接、拆解日期、赛道、标题公式、Hook模式、情绪触发、结构类型、适合选题、适合目标号、爆文级别、参考价值、金句摘录、可复用标题模板、拆解备注等）。

### Step 5: 直接使用

用户发一篇公众号文章URL，AI抓取全文、分析结构、自动写入飞书表格17个字段。

### Step 6: 自动修Bug

用户发现重复数据：
> 为啥一篇文章会有2个记录

AI自动：查表格 → 删重复记录 → 检查技能文件逻辑 → 修bug（加写入前查重）→ 约定以后覆盖更新。

## 关键认知

1. 用户全程没有写过一行命令
2. 多维表格的表结构是AI设计的，不是用户
3. 用户只需要：发链接、点授权、说需求
4. AI负责：安装、配置、建表、写入、查重、修bug、迭代

## 对话式操作的核心原理

- lark-cli 安装：`npm install -g lark-cli`（AI执行）
- 配置：AI用 `lark-cli config init` 处理凭证
- 认证：`lark-cli auth login` 生成授权链接，用户浏览器点确认
- 建表：AI调用飞书API自动创建多维表格和字段
- 写入：`lark-cli base +record-search` 查重 → `lark-cli base +record-upsert` 创建/更新
- 所有命令由AI翻译用户自然语言来执行
